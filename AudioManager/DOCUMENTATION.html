Audio System Documentation

A powerful and flexible audio management system for Unity, featuring sound collections, individual sounds, and advanced audio control capabilities.

Getting Started

First, make sure to include the Audio System namespace in your scripts:

Namespace
using UnityEngine;
using HawaHawa.AudioSystem;

The Audio System provides three main ways to manage sounds in your Unity project:

  • AudioCollection - Manage groups of related sounds
  • SingleSound - Control individual sound effects
  • SoundFromGroup - Access specific sounds within collections
Example Fields
public AudioCollection musicCollection;    // For managing background music
public SingleSound jumpSound;           // For a specific sound effect
public SoundFromGroup footsteps;        // For accessing a specific sound in a group

Inspector Assignment vs Runtime Access

Audio components can be assigned in two ways:

  • Using [SerializeField] or public fields for inspector assignment
  • Using Get functions for runtime access when fields are private or need to be accessed dynamically
Inspector Assignment
[SerializeField] private AudioCollection musicCollection;    // Visible in inspector
[SerializeField] private SingleSound jumpSound;           // Visible in inspector
[SerializeField] private SoundFromGroup footsteps;        // Visible in inspector

Get Functions

When you need to access audio components at runtime or can't assign them through the inspector, use these Get functions:

AudioSystem.GetAudioCollection(string collectionName)

Retrieves an audio collection by its name. Use this for managing groups of related sounds like background music, ambient sounds, or character voice lines.

AudioSystem.GetSingleSound(string soundName)

Retrieves an individual sound by its name. Perfect for one-off sound effects that don't belong to a collection.

AudioSystem.GetSoundFromGroup(string groupName, string soundName)

Retrieves a specific sound from a named group. Useful when you need direct access to one sound within a collection.

Runtime Access
AudioCollection musicCollection = AudioSystem.GetAudioCollection("BackgroundMusic");
SingleSound jumpSound = AudioSystem.GetSingleSound("PlayerJump");
SoundFromGroup footsteps = AudioSystem.GetSoundFromGroup("PlayerSounds", "Footsteps");

Single Sound

Use SingleSound for managing standalone sound effects that aren't part of a collection.

Creating a Single Sound

SingleSound Example
public SingleSound singleSound;

Available Functions

singleSound.Play() - Plays the individual sound with default settings.

singleSound.Play(float volume, float pitch) - Plays the sound with custom volume and pitch settings.

singleSound.PlayRandomPitch(float minPitch, float maxPitch) - Plays the sound with random pitch between min and max values.

singleSound.Stop() - Stops playback of the sound.

singleSound.FadeIn(float duration, float targetVolume) - Gradually increases volume over duration to target volume.

singleSound.FadeOut(float duration) - Gradually decreases volume over duration.

singleSound.Pause() - Pauses playback of the sound.

singleSound.Resume() - Resumes playback of a paused sound.

Common Use Cases

  • One-shot Sound Effects: Jump sounds, attack impacts, item pickups
  • UI Feedback: Button clicks, notification alerts, error sounds
  • Character Actions: Dash sounds, special ability activations
  • Environmental Triggers: Door opening/closing, lever pulls, switches
  • Achievement Unlocks: Single celebration sounds, level completion jingles
  • Game Events: Countdown timers, round start/end signals
  • Menu Navigation: Selection confirms, back button sounds
  • Status Changes: Health pickup, power-up activation, shield break

Audio Collections

Audio Collections are containers that manage multiple audio clips as a group. They provide functionality for playing, stopping, and manipulating multiple related sounds from a single container. Each collection can hold multiple audio clips and provides methods for both specific and random playback.

Creating an Audio Collection
public AudioCollection sfxCollection;

Available Functions

sfxCollection.Play(string soundName) - Plays a specific sound from the collection using its name.

sfxCollection.Play(string soundName, float volume, float pitch) - Plays a sound with custom volume (0-1) and pitch settings.

sfxCollection.PlayRandomPitch(string soundName, float minPitch, float maxPitch) - Plays a specific sound with random pitch between min and max values.

sfxCollection.Stop(string soundName) - Stops playback of a specific sound in the collection.

sfxCollection.PlayRandomAudio() - Plays a random sound from the entire collection.

Common Use Cases

  • Background Music Collections: Managing different tracks for various game states or levels
  • Character Sound Effects: Grouping voice lines, footsteps, or action sounds
  • UI Sound Collections: Managing button clicks, hover sounds, and menu transitions
  • Ambient Sound Groups: Organizing environmental sounds like weather, nature, or atmosphere
  • Combat Sound Effects: Grouping weapon sounds, impact effects, or ability sounds
  • Vehicle Sound Collections: Managing engine sounds, horns, or impact effects
  • NPC Voice Collections: Organizing dialogue or reaction sounds for characters
  • Achievement/Reward Sounds: Managing positive feedback sounds for player accomplishments

Sounds from Groups

SoundFromGroup provides direct access to specific sounds within collections, offering granular control over individual sounds while maintaining the organizational benefits of collections.

Creating a Sound from Group
public SoundFromGroup groupSound;

Available Functions

groupSound.Play() - Plays the specific sound from its group with default settings.

groupSound.Play(float volume, float pitch) - Plays the sound with custom volume (0-1) and pitch settings.

groupSound.PlayRandomPitch(float minPitch, float maxPitch) - Plays the sound with random pitch between min and max values.

groupSound.Stop() - Stops playback of the sound.

groupSound.FadeIn(float duration, float targetVolume) - Gradually increases volume over duration to target volume.

groupSound.FadeOut(float duration) - Gradually decreases volume over duration.

groupSound.Pause() - Pauses playback of the sound.

groupSound.Resume() - Resumes playback of a paused sound.

Common Use Cases

  • Character Voice Lines: Managing specific dialogue options from a character's voice collection
  • Weapon Variations: Controlling individual weapon sounds from a weapons sound collection
  • Music Tracks: Managing specific background music tracks from a music collection
  • Environmental Effects: Controlling individual ambient sounds from an environment collection
  • Vehicle Components: Managing specific vehicle sounds (engine, horn, brakes) from a vehicle sound collection
  • UI Element Sounds: Controlling specific UI sounds from a UI sound collection
  • Footstep Variations: Managing specific footstep sounds for different surfaces
  • Impact Effects: Controlling specific impact sounds from a collection of collision effects

Best Practices

  • Use AudioCollections for related sounds (e.g., footsteps, UI sounds)
  • Use SingleSound for unique, standalone effects
  • Implement FadeIn/FadeOut for smooth transitions
  • Use PlayRandomPitch for natural variation in repetitive sounds
  • Group similar sounds in collections for better organization

Example Usage

AudioManager.cs
using UnityEngine;
using HawaHawa.AudioSystem;

public class AudioManager : MonoBehaviour
{
    // Audio components
    [SerializeField] private SingleSound playerJumpSound;
    [SerializeField] private AudioCollection musicCollection;
    [SerializeField] private SoundFromGroup footstepSound;

    private void Update()
    {
        // SingleSound example - Space key for jump sound
        if (Input.GetKeyDown(KeyCode.Space))
        {
            playerJumpSound.PlayRandomPitch(0.9f, 1.1f);
        }

        // AudioCollection example - M key for music control
        if (Input.GetKeyDown(KeyCode.M))
        {
            musicCollection.Play("MainTheme", 0.8f, 1.0f);
        }
        else if (Input.GetKeyDown(KeyCode.N))
        {
            musicCollection.Stop("MainTheme");
        }

        // SoundFromGroup example - F key for footstep with fade
        if (Input.GetKeyDown(KeyCode.F))
        {
            footstepSound.FadeIn(0.5f, 1.0f);
        }
        else if (Input.GetKeyUp(KeyCode.F))
        {
            footstepSound.FadeOut(0.3f);
        }

        // Additional controls
        if (Input.GetKeyDown(KeyCode.R))
        {
            // Play random sound from collection
            musicCollection.PlayRandomAudio();
        }

        if (Input.GetKeyDown(KeyCode.P))
        {
            // Pause/Resume footstep sound
            if (Input.GetKey(KeyCode.LeftShift))
                footstepSound.Resume();
            else
                footstepSound.Pause();
        }
    }
}

Key Controls

  • Space - Plays jump sound with random pitch variation
  • M - Plays main theme from music collection
  • N - Stops main theme
  • F (Hold) - Fades in footstep sound, fades out when released
  • R - Plays a random sound from the music collection
  • P - Pauses footstep sound
  • Shift + P - Resumes footstep sound
